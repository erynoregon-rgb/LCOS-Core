# Artifact-Backed AI Agent Governance and Workflow Audit

## One-line position

I help teams make AI-assisted work auditable from artifacts instead of summaries.

## What I do

I build and audit workflows for tool-using AI agents, coding assistants, and multi-agent development systems. My focus is not just whether an AI system produces useful output, but whether the work can be bounded, routed, recorded, tested, reviewed, replayed, and corrected.

## Problems I can help with

AI-assisted teams often run into hidden workflow failures:

- agents say "done" while files remain uncommitted, untested, or unwired
- tools are built but never connected to the live path
- local tests are mistaken for full proof
- dirty repo/runtime state becomes impossible to explain
- prompts, summaries, and tickets drift away from actual repo state
- high-risk changes lack recovery plans or proof floors
- documentation claims more than the system actually supports
